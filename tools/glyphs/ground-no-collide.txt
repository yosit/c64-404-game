// Ground design (redrawn 2026-07-04). The land strip is 3 char-rows (12/13/14);
// only the BOTTOM char row (row 14, screen y162-169) may hold pixels, because
// the running dino's lowest pixel is y161 — keeping all ground art at y>=162
// means $d01f (sprite<->background) never fires on plain ground, so a set bit
// always means "hit a cactus". Pixel-perfect, like Chrome.
//
// So: the top+middle char rows of the plain-ground tiles (1,2,4) are blanked
// (clean sky), and the bottom char row draws a crisp continuous ground line
// (top pixel = y162, right under the dino's feet) with sparse pebbles/tufts.
//
// Tile n occupies chars n*9 .. n*9+8, row-major 3x3:
//   top    = base+0,1,2     middle = base+3,4,5     bottom = base+6,7,8
// Plain-ground tiles in use (screen.asm LandTiles): 1, 2, 4.
